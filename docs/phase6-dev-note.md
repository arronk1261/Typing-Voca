# Phase 6 개발 노트 — 콘텐츠 정제 (학습 문장 품질 다지기)

> 비개발자(기획자)도 "사용자가 무엇을 보고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 6에서 무엇을 했나

Phase 0~5로 **학습 앱의 모든 기능**(타이핑→섀도잉→통계→리포트)은 완성됐습니다.
Phase 6은 기능이 아니라 **"학습자가 외우는 문장 그 자체의 품질"** 을 다지는 단계입니다. 아무리 화면이 예뻐도 **외우는 영어 문장이 어색하거나 틀려 있으면** 학습 서비스로서 신뢰가 무너지기 때문입니다.

비유하자면, 멋진 교실과 칠판(앱)을 다 지었으니 이제 **교과서의 오탈자와 어색한 예문을 교정하는** 작업입니다.

- **🧹 단어 DB 대확충 (900 → 2,520문항)**: 레벨 3개에 900개로는 "일상 회화"를 다 다루기 어려워, **12개 주제 × 3레벨 × 70개 = 2,520문장**으로 늘렸습니다(이미 완료·적재됨).
- **🔖 문장마다 "꼬리표(태그)" 부착**: 각 문장에 *얼마나 자주 쓰이나(빈도)*, *단어인가 숙어인가(유형)*, *어떤 상황에서 쓰나(use_case)* 같은 정보표를 붙였습니다. 나중에 "쉬운 것부터 똑똑하게 출제"하는 데 씁니다.
- **🔍 자동 교정기(검수 스크립트)**: 2,520문장을 **사람이 일일이 보지 않아도** 규칙 위반을 잡아내는 자동 검사기를 만들었습니다.
- **♻️ 중복 문장 정리**: 같은 표현이 여러 주제에 나올 때, *학습에 좋은 반복*과 *그냥 복붙된 중복*을 갈라내 후자만 손봤습니다.

> 핵심 한 줄: **"기능"이 다 된 앱에, 이제 "믿고 외울 수 있는 콘텐츠"라는 알맹이를 채웠다.**

## 2. 가장 중요한 장치 — "자동 교정기" 두 개

사람이 2,520문장을 눈으로 검사하면 반드시 실수가 새어 나갑니다. 그래서 **버튼 한 번이면 전수 검사하는 자동 교정기 두 개**를 만들었습니다.

### ① 콘텐츠 검수기 (`npm run validate:words`)
교과서 전체를 훑으며 아래를 자동으로 잡아냅니다.
- **빈칸을 채우면 음성과 똑같은가?** — 학습 화면은 `___`(빈칸)을 보여주고, 정답을 넣으면 완성 문장이 됩니다. 이 완성 문장이 **읽어주는 음성(TTS) 문장과 글자 하나까지 일치**하는지 검사. (안 맞으면 "들리는 말 ≠ 보이는 말"이 되어 학습이 깨짐)
- **빈칸이 정확히 1개인가?**
- **정답 칸에 'someone' 같은 빈 자리표시가 새어 들어가지 않았나?** (예전 900문항 시절 `ping someone you` 같은 어색한 잔재)
- **한 주제 안에 똑같은 정답이 두 번 나오지 않나?**
- **주제·레벨별로 정확히 70문장씩 채워졌나?**

> 결과: **2,520문장 전수 검사 → 오류 0건.** (경고 6건이 떴지만, 모두 "Are you hungry? Let's grab **something**." 처럼 *정답 칸이 아니라 자연스러운 전체 문장 속 단어*라 정상입니다.)

### ② 중복 분석기 (`npm run dedup:report`)
"같은 표현이 여러 주제에 등장하는 것"을 **좋은 반복**과 **나쁜 복붙**으로 자동 분류합니다.

- ✅ **좋은 반복(그대로 유지)**: 예를 들어 "drop by(잠깐 들르다)"가 *인사* 주제에선 "근처 오면 들러요", *약속* 주제에선 "언제든 들러도 돼요"처럼 **서로 다른 예문**으로 나오는 것. → 같은 표현을 다른 맥락에서 다시 만나면 오히려 **기억에 더 잘 남습니다**(간격 반복).
- ❌ **나쁜 복붙(교체 대상)**: 주제만 다를 뿐 **예문이 글자까지 똑같은** 경우. 맥락 차별화가 없어 학습 가치가 없습니다.

> 결과: 카테고리 간 중복 238종 중 **"글자까지 똑같은 복붙" 9종**을 찾아내, 각 주제 맥락에 맞는 새 예문으로 바꿨습니다. 재검사 시 **나쁜 복붙 0종.** 모든 판정 사유는 `docs/duplicate-answer-report.md`에 자동 기록됩니다.

> 비유: 검수기는 **맞춤법 검사기**, 중복 분석기는 **"이 문장 어디서 베꼈나" 표절 검사기**입니다. 두 검사기는 앞으로 콘텐츠를 추가할 때마다 다시 돌려 **품질이 다시 나빠지는 걸 막는 안전망**이 됩니다.

## 3. 실제로 고친 9문장 — 무엇을 어떻게 바꿨나

예문이 글자까지 똑같던 중복 9건을, **주제(카테고리)에 어울리는 새 문장**으로 바꿨습니다. (정답 표현 자체는 그대로 두고, 그것이 등장하는 *상황 문장*만 차별화)

| 표현 | 한쪽은 그대로 두고… | 다른 쪽을 이렇게 바꿈 (주제에 맞게) |
|---|---|---|
| drop by (들르다) | "Feel free to drop by anytime." (약속) | **"Why don't you drop by after work?"** (친목) |
| miss (그립다/놓치다) | "I really miss you." (감정) | **"I miss hanging out with you."** (친목) |
| break (휴식) | "Let's take a short break." (직장) | **"There's a short break between meetings."** (일정) |
| follow up (다시 확인하다) | "I'll follow up with you tomorrow." (직장) | **"Let's follow up next week."** (일정) |
| mind (신경 쓰다) | "Do you mind if I sit here?" (인사) | **"I don't mind at all."** (감정) |
| pay by card (카드 결제) | "Can I pay by card?" (쇼핑) | **"We'd like to pay by card."** (식당) |
| printer (프린터) | "The printer is out of ink." (직장) | **"I can't connect to the printer."** (IT) |
| visit (방문하다) | "Come visit us sometime." (인사) | **"I'd love to visit your new place."** (친목) |
| wear one's heart on one's sleeve (감정을 드러내다) | "He tends to wear his heart on his sleeve." (감정) | **"Among friends, he tends to wear his heart on his sleeve."** (친목) |

> 이렇게 하면 학습자는 같은 표현을 **두 가지 다른 상황**에서 만나, 단순 암기가 아니라 **"언제 쓰는 말인지"** 까지 익히게 됩니다.

## 4. 화면 뒤 작업 흐름 — 안전하게 고치는 파이프라인

문장을 손으로 직접 고치면 실수(빈칸-음성 불일치 등)가 나기 쉽습니다. 그래서 **공장 컨베이어처럼** 정해진 순서로만 고쳤습니다.

1. **원본 수정**: 주제별 원본 파일(`generated/*.json`)의 9문장을 패치. (최종 파일이 아니라 *원천*을 고쳐야 나중에 다시 만들어도 유지됨)
2. **재조립**(`npm run assemble:words`): 원본들을 모아 **번호를 새로 매기고 + 검수기로 자동 검증**하며 최종 파일(`src/data/words.json`)을 다시 만듭니다.
3. **재검사**(`validate:words` + `dedup:report`): 오류 0 / 나쁜 복붙 0 확인.
4. **클라우드 반영**(`npm run seed`): 고친 2,520문장을 Supabase(서버 DB)에 다시 올림.
5. **백업본 동기화**: 프로젝트 루트의 `words.json`·`words.csv`(엑셀용)도 최신으로 맞춤.

> 결과: **콘텐츠 단일 소스가 3곳(앱 내부 / 루트 백업 / 클라우드 서버) 모두 똑같이 일치.** 어디서 봐도 같은 문장입니다.

## 6. 마스터 브리핑 (한 문단 요약)

> Phase 6은 "보이는 기능"이 아니라 **"믿고 외울 수 있는 콘텐츠"** 를 다진 단계입니다. 학습 문장을 900개에서 **2,520개**로 늘려 일상 회화 전 영역을 덮었고, 문장마다 빈도·유형·상황 **꼬리표**를 달아 앞으로 똑똑한 출제의 토대를 놓았습니다. 무엇보다 사람 눈 대신 **두 개의 자동 검사기**(맞춤법 검사기 격인 콘텐츠 검수기, 표절 검사기 격인 중복 분석기)를 만들어, **2,520문장 전수 오류 0건 / 나쁜 복붙 0건**을 못 박았습니다. "글자까지 똑같던" 9문장은 각 주제에 어울리는 새 예문으로 바꿔, 학습자가 같은 표현을 여러 상황에서 자연스럽게 다시 만나도록 했습니다. 이 검사기들은 앞으로 콘텐츠를 추가할 때마다 다시 돌려 **품질이 무너지지 않게 지키는 안전망**으로 남습니다.
>
> **결과: 잘 만든 앱(0~5)에, 신뢰할 수 있는 알맹이(콘텐츠 품질)가 채워졌습니다. 다음은 Phase 7(레벨 진단·채점 신뢰도)로 이 콘텐츠 태그를 활용할 차례입니다.**
