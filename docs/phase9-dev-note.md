# Phase 9 개발 노트 — 신뢰도·무중단·적응 보강 (비개발자용)

> 외부 코드리뷰(Codex)가 짚어준 점들을 반영한 단계예요. 한 줄 요약: **"발음 연습이 안 되는 환경에서도 단어를 끝까지 외울 수 있게" 고치고, 레벨 진단과 출제·승급을 더 똑똑하게 다듬었습니다.** 이번엔 데이터베이스 구조를 바꾸지 않아서, 별도 설정 작업 없이 배포만 하면 적용돼요.

---

## 🔴 가장 중요한 수정 — "발음이 안 되면 단어를 영영 못 외우던" 문제

### 무엇이 문제였나
이 앱은 한 단어를 외울 때 **① 타이핑으로 빈칸 채우기 → ② 따라 말하기(발음)** 두 단계를 거쳐요. 그런데 휴대폰/브라우저가 마이크 인식(STT)을 지원하지 않으면 ② 발음 단계가 자동으로 생략됩니다(이걸 "타이핑 전용 모드"라고 불러요).

문제는, 단어를 "다 외웠다(졸업)"고 인정하는 조건이 **"타이핑 정답 + 발음 별점 2개 이상"** 이었다는 점이에요. 발음 단계가 아예 없는 환경에서는 별점을 받을 수 없으니, **타이핑을 아무리 완벽하게 해도 영원히 졸업이 안 되고**, 심지어 그동안 쌓아온 통과 기록까지 0으로 초기화됐습니다.

이건 이 프로젝트의 1순위 원칙인 **"학습은 어떤 환경에서도 멈추지 않는다"** 를 정면으로 어기는 거였어요.

### 어떻게 고쳤나
- **타이핑 전용 모드에서는 타이핑만 잘해도 졸업**할 수 있게 했어요(발음은 그 환경에 없으니 빼고 인정).
- 사용자가 발음을 **스스로 건너뛴 경우**는, 졸업으로 쳐주지도 않지만 **불이익도 주지 않아요**(기존 통과 기록을 깎지 않음). "발음 점수가 없다"는 이유만으로 벌점을 주던 동작을 없앴습니다.
- 진짜 실패(타이핑에서 하트를 다 소진했거나, 발음을 시도했는데 많이 어긋난 경우)일 때만 복습으로 돌아갑니다.

> 사용자 입장: 마이크가 안 되는 폰에서도, 발음을 건너뛰어도, **학습이 막히거나 그동안의 노력이 사라지지 않아요.**

## 🟡 레벨 진단·출제를 더 똑똑하게

### 레벨 테스트가 더 "진단답게"
- 기존엔 테스트 문제가 쉬운 단어 위주로, 카테고리도 한쪽에 몰려 있었어요. 이제 **레벨별로 "회화 실력이 드러나는 표현"**(연어·구동사·관용구 같은 덩어리 표현)을 골고루 섞고, **주제도 11개로 분산**했습니다.
- 결과 화면도 "Lv.2"만 딱 보여주지 않고, **"단어 실력은 탄탄해요! 덩어리 표현을 조금만 더 연습하면 회화가 트여요"** 처럼 **강점·약점을 알려주는 격려 메시지**를 함께 보여줘요.

### 오늘의 단어 구성 개선
- **너무 niche한 표현**(예: 'prix fixe', 'corkage fee' 같은 식당 전문 용어)이 기본 학습에 과하게 끼지 않도록 비율을 제한했어요.
- **카테고리 학습**은 단순히 그 주제 단어를 무작위로 뽑는 대신, **상황 흐름 순서**(예: 여행 = 공항 → 교통 → 길찾기 → 호텔 …)로 정렬해 **미니 시나리오**처럼 흐르게 했습니다.

## 🟢 여러 기기에서 일관된 학습 + 콘텐츠 심화

- **승급 판단의 기기 간 일관성**: "레벨을 올려볼까요?" 제안은 최근 학습 흐름을 보고 결정하는데, 예전엔 이 기록이 **그 기기 안에만** 저장됐어요. 이제 **클라우드(서버)에 저장된 최근 학습 세트**를 불러와 판단하므로, **휴대폰에서 하다 노트북에서 이어 해도** 판단이 끊기지 않아요.
- **커리큘럼 단계 구분**: 단어를 다시 일일이 분류하지 않고, 기존 정보(레벨·빈도·표현 유형)에서 **생존회화 / 일상 / 업무 / 심화** 단계를 자동으로 추정해, **초보자에게는 생존·일상 표현을 먼저** 보여줍니다.
- **발음 포커스**: 따라 말하기에서 잘 안 됐던 단어를 모아, 세션 결과 화면에 **"🎤 발음 포커스 — 다음엔 이 단어를 또렷하게!"** 로 콕 집어줘요(th, r/l 같은 한국인이 어려워하는 발음을 우선 표시).
