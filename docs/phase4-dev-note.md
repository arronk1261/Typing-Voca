# Phase 4 개발 노트 — 결과창 · 복습 졸업 · 세션 기록 · 출제 로직

> 비개발자(기획자)도 "사용자가 무엇을 보고, 무엇을 누르고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 4에서 무엇을 만들었나

Phase 1~3은 **"한 문항을 푸는 경험"**(타이핑 → 듣고 따라 말하기)이었습니다.
Phase 4는 그 10문항을 **"한 세트의 학습"으로 마무리하고, 기록으로 남기고, 다음 세트를 똑똑하게 골라주는** 단계입니다.

- **🏆 결과창**: 한 세트(10문항)를 끝내면 오늘 성적표가 뜹니다. 학습 단어 수, 첫 시도 정답, 평균 별점, 복습 노트 개수 + 단어별 결과 + 축하 색종이.
- **🔁 복습 졸업 규칙(SRS-lite)**: 틀린 단어는 "복습 노트"에 들어가고, **두 번 잘 통과하면 졸업**합니다. 한 번 맞혔다고 끝이 아니라, 진짜 외울 때까지 다시 나옵니다.
- **🎯 출제 로직**: 다음 세트를 **새 단어 7개 + 복습 3개** 비율로, 복습은 "다시 볼 때가 된 것" 우선으로 자동 편성합니다.
- **📈 적응형 레벨 제안**: 너무 쉬우면 "레벨 올려볼까요?", 너무 어려우면 "한 단계 낮춰도 좋아요" 하고 **부드럽게 제안**(강요 아님).
- **💾 세션 기록 저장**: 세트가 끝날 때마다 요약 한 줄을 클라우드에 저장 → Phase 5의 통계·리포트 재료가 됩니다.

> 핵심 한 줄: **"한 문항"이 모여 "한 세트의 학습"이 되고, 그게 기록으로 쌓여 "내 성장 곡선"이 된다.**

## 2. 가장 중요한 설계 — "딱 한 번, 길목에서만 저장한다"

우리 앱의 절대 규칙 중 하나는 **"키 입력·문항마다 클라우드를 두드리지 않는다"**입니다(느리고 데이터 낭비라서).
그래서 학습하는 내내 결과는 **머릿속(메모리)** 에만 쌓아두다가, **세트가 끝나는 길목(결과창)에서 단 한 번** 클라우드에 몰아 저장합니다. 시험을 다 본 뒤 성적을 한 번에 적는 것과 같습니다.

이 "한 번에 몰아 저장" 담당이 이번에 만든 **`commitSession`**(세션 결산원)입니다. 결과창이 뜨는 순간 이 결산원이 딱 한 번 호출되어:

1. **단어별 학습 신호** 갱신 (복습 졸업 규칙 적용)
2. **연속 학습일(streak)·누적 단어 수** 갱신
3. **세션 요약 한 줄** 기록

을 **한 묶음으로** 처리합니다.

> ⚠️ **개발 중 잡은 버그 한 토막 (이해를 돕는 사례)**: 개발 모드에서는 React가 안전성 점검을 위해 화면 준비 동작을 **일부러 두 번** 실행합니다. 그 탓에 결산원이 두 번 호출돼 누적 단어가 10이 아니라 **20**으로 찍히는 현상을 발견했습니다. → "이미 결산했나?"를 **저장소에 즉시 깃발로 꽂아** 두 번째 호출을 막아 해결했습니다. (성적표를 두 번 제출하지 않도록 도장을 찍는 것과 같습니다.) 실제 배포 모드에선 두 번 실행이 없지만, 안전하게 양쪽 다 막았습니다.

## 3. 화면 뒤에서 벌어지는 일 — 단계별 원리

### (4-1) 결과창 — 오늘의 성적표 + 축하

- 10문항을 끝내면 트로피와 **색종이(컨페티)** 가 터지고, 4개 숫자(학습 단어 / 첫 시도 정답 / 평균 별점 / 복습 노트)와 단어 목록이 뜹니다.
- 단어별 아이콘으로 한눈에: **⭐⭐⭐**(섀도잉 별점) / **⏭**(발음 건너뜀) / **📒**(복습 노트행) / **✓**(통과).
- 이때 결산원이 호출되어 **연속 학습일 불꽃이 강해지고**, 모든 기록이 클라우드로 갑니다.
- 아래 버튼 3종: **[방금 틀린 단어 복습 (N)]**(이번에 어려웠던 단어만 모아 즉시 복습), **[한 세트 더]**, **[메인 로비]**.

### (4-2) 복습 졸업 규칙(SRS-lite) — "진짜 외울 때까지"

가장 머리를 많이 쓴 부분입니다. 단어 하나하나에 대해 이런 규칙이 돕니다(담당: `lib/srs.ts`):

| 상황 | 처리 |
|---|---|
| **복습 노트에 들어감** | 타이핑 목숨(하트)을 다 쓰거나, 따라 말하기 별점이 ⭐ 1개 이하일 때 |
| **"통과" 1회** | 타이핑을 **첫 시도에 맞히고** + 따라 말하기 **⭐⭐ 이상**일 때 |
| **1차 통과** | 내일(+1일) 다시 보도록 예약 |
| **2차 통과** | **졸업** — 복습 노트에서 빠짐 |
| **통과 못 함** | 통과 횟수를 0으로 리셋(단어는 노트에 그대로 유지) |

> 쉽게: **"한 번 잘했다고 끝이 아니라, (다른 기회에) 두 번 연속 잘해야 졸업."** 진짜 장기 기억으로 넘어갈 때까지 붙잡아 둡니다.
> 이 규칙은 **자동 테스트 7개**(`scripts/test-srs.ts`)로 "진입·통과·졸업·리셋"이 정확히 도는지 못 박아 두었고, 전부 통과합니다.

### (4-3) 출제 로직 — 다음 10문항을 똑똑하게 고르기

다음 세트를 만들 때(담당: `buildSession`):

- **새 단어 7 : 복습 3** 비율로 섞습니다.
- **새 단어**는 "아직 안 본 것" 먼저, 그다음 "오래전에 본 것".
- **복습**은 "다시 볼 때(next_due)가 지난 것"을 오래된 순으로.
- **카테고리 학습**이면 그 주제로 거르고, 부족하면 인접 레벨에서 채웁니다.
- **한 세션 안에서 같은 단어는 두 번 안 나옵니다.**

### (4-3) 적응형 레벨 제안 — 강요하지 않는 코치

세트 성적을 보고 살짝 귀띔만 합니다(`suggestLevelAdjustment`):
- 첫 시도 정답률이 높고(80%+) 따라 말하기 평균이 ⭐⭐ 이상이면 → *"정말 잘하고 있어요! 다음엔 Lv.N에 도전해볼까요?"* (토스트)
- 하트 소진이 잦으면(40%+) → *"조금 버거웠나요? Lv.N부터 다져도 좋아요."*
- **절대 강제로 레벨을 바꾸지 않습니다.** 빨강·경고 톤도 쓰지 않습니다(T2: 항상 긍정 강화).

### (4-4) 세션 기록 — 통계의 씨앗

세트가 끝나면 요약 한 줄을 `study_sessions`에 남깁니다: 레벨, 카테고리, 문항 수, 첫 시도 정답 수, 평균 별점, 평균 점수, 복습 개수, 날짜.
- 이 한 줄들이 쌓여 **Phase 5의 스트릭 캘린더·학습 추이·주간 리포트**가 됩니다.
- **인터넷이 끊겨도** 기록은 기기 안 대기열(큐)에 저장됐다가, 연결되면 자동으로 올라갑니다 → **학습 루프는 절대 끊기지 않음**(규칙 6).

## 4. 오프라인·게스트는 어떻게 처리되나

| 상황 | 동작 |
|---|---|
| **로그인 + 온라인** | 결과를 클라우드(Supabase)에 즉시 1회 배치 저장. 다른 기기에서도 이어짐. |
| **로그인 + 오프라인** | 변경분을 기기 대기열에 적재 → 인터넷 복귀 시 자동 업로드. |
| **게스트(로그인 안 함)** | 클라우드엔 안 쓰지만, 메모리에 누적되어 그 세션 흐름(복습 노트 등)은 정상 동작. |

> 어떤 경우에도 **"저장이 안 돼서 학습이 멈추는 일"은 없습니다.**

## 5. 만든 파일·바뀐 파일 한눈에

**새로 만든 것**
- `lib/srs.ts` — 복습 졸업 규칙 엔진(진입·통과·졸업·리셋, 다시 볼 날 계산).
- `scripts/test-srs.ts` + `scripts/alias-resolve.mjs` + `scripts/test-bootstrap.mjs` — SRS 자동 검증 테스트와 그 실행 도우미.

**손본 것**
- `stores/userStore.ts` — **`commitSession`**(세션 결산원) 추가: 진도·스트릭·누적·세션요약을 한 번에 계산·저장.
- `lib/words/getWords.ts` — **`buildSession`**(신규70/복습30 출제) + **`suggestLevelAdjustment`**(레벨 제안) 추가.
- `components/study/SessionResult.tsx` — 결산 호출·평균 별점·[방금 틀린 단어 복습] 버튼·레벨 제안 토스트, 그리고 **중복 결산 방지** 처리.
- `components/study/StudySession.tsx` — 출제를 진도(progress) 기반 `buildSession`으로 교체.
- `stores/sessionStore.ts` / `types/index.ts` — `committed`(결산 완료 깃발) 등 보강.

## 6. 다음 Phase에 넘기는 재료

이제 매 세트마다 **요약 한 줄(study_sessions)** 과 **단어별 학습 신호(progress)** 가 차곡차곡 쌓입니다.
**Phase 5**는 이 데이터를 읽어 **스트릭 캘린더, 학습량·점수 추이 그래프, 카테고리별 숙련도, 주간 리포트**를 그립니다.

> 한 줄 요약: **Phase 4에서 "학습이 끝나고 → 기록되고 → 다음이 똑똑하게 준비되는" 루프가 완성됐다.**

## 마스터 브리핑 (사용자 여정으로)

> 사용자가 10문항을 끝냈습니다. **트로피가 튀어 오르고 색종이가 쏟아지며** 오늘의 성적표가 뜹니다 —
> 학습 단어 10개, 첫 시도 정답 6개, 평균 ⭐2.1, 복습 노트 2개. 화면이 *"정말 잘하고 있어요! 다음엔 Lv.2에
> 도전해볼까요?"* 라고 살짝 귀띔합니다(강요는 없습니다). 그동안 화면 뒤에서는 **결산원이 단 한 번** 움직여,
> 어려웠던 2개를 복습 노트에 넣고, 잘한 단어의 졸업 카운트를 올리고, 연속 학습일 불꽃을 키우고, 오늘 요약을
> 클라우드에 적었습니다. 사용자가 [방금 틀린 단어 복습]을 누르면 그 2개만 모아 바로 다시 도전하고,
> [한 세트 더]를 누르면 **새 단어 7 + 복습 3**으로 똑똑하게 짜인 다음 세트가 시작됩니다. 인터넷이 잠깐
> 끊겨 있었다면? 기록은 대기열에 안전하게 보관됐다가 연결되는 순간 조용히 올라갑니다.
>
> **결과: 한 세트의 학습이 "결과 확인 → 복습 졸업 관리 → 기록 → 다음 출제"로 매끄럽게 순환하게 되었습니다.**
