# Phase 9-1 개발 노트 — 주간 발음 리포트 + 발음 전수 분석 (비개발자용)

> Phase 9에서 "효과 대비 작업량이 커서 잠시 미뤄둔" 두 가지를 마저 구현한 단계예요. 한 줄 요약: **"내 발음 약점이 한 주 단위로 쌓여 리포트로 보이고, 모든 단어의 발음 난이도를 자동으로 분석"** 합니다.

---

## 1) 🎤 이번 주 발음 약점 리포트 (9-1a)

### 무엇이 달라졌나
따라 말하기(섀도잉)를 하면 시스템이 "이 단어가 잘 안 들렸어요" 하고 **약했던 단어**를 골라내요. 예전엔 이 정보가 **그 세션이 끝나면 사라졌어요**. 이제는 **클라우드에 차곡차곡 저장**되어서, **주간 리포트 화면**에 모아 보여줍니다.

- 주간 리포트(`/report`)에 **"이번 주 발음 약점"** 카드가 새로 생겼어요.
- 약점 단어를 **자주 틀린 순 + 발음 난이도 순**으로 정렬해 보여줍니다(예: `the ×3`, `world ×2`).
- 단어들이 어떤 발음에 몰려 있는지도 짚어줘요 — 예: **"특히 th 발음, r/l 구분을 또렷하게 연습해 보세요 🎤"**

> 사용자 입장: 한 주를 돌아보며 "아, 내가 이번 주엔 th 발음이 약했구나" 하고 **구체적인 연습 포인트**를 얻습니다.

### 참고 — 설정 작업 1회 필요
이 기능은 학습 기록에 "약점 단어" 칸을 새로 추가해서 저장해요. 그래서 **Supabase에서 `v9_session_weak_words.sql`을 한 번 실행**해야 클라우드에 저장됩니다(이전 v7·v8과 같은 방식).
- 아직 실행 전이어도 **앱은 멈추지 않아요**(저장만 잠시 생략) — 실행하면 그때부터 리포트가 채워집니다.

## 2) 🔤 발음 난이도 전수 분석 (9-1b)

### 무엇이 달라졌나
예전엔 "발음이 어려운 단어"로 표시된 게 **2,520개 중 23개**뿐이었어요. 이제 **모든 단어를 자동으로 분석**해서, 한국인이 어려워하는 발음 요소가 들어있는지 표시합니다.

- 분석 대상 발음 요소: **th**(the/think), **r/l 구분**, **v/f**, **z**, **자음군**(str·spl 같은 연속 자음).
- 결과: **2,520개 중 1,363개(54%)** 가 한 가지 이상 까다로운 발음 요소를 포함 — 23개에서 대폭 확대됐어요.
- 이 분석 결과는 위 **주간 발음 리포트의 "어떤 발음이 약한지"** 설명에 그대로 쓰입니다.

### 어떻게 만들었나 (가볍게)
단어를 일일이 손으로 다시 태깅하지 않고, **글자 패턴에서 자동으로 판별**하는 방식이에요. 그래서 데이터베이스를 다시 채우는 무거운 작업 없이도 앱이 실시간으로 같은 판별을 합니다. 전체 분포는 `docs/pronunciation-coverage.md`에 리포트로 정리해 뒀어요(`npm run tag:pronunciation`로 재생성).
