# Phase 5 개발 노트 — 통계 히스토리 · 주간 리포트 · 최종 QA

> 비개발자(기획자)도 "사용자가 무엇을 보고, 무엇을 누르고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 5에서 무엇을 만들었나

Phase 4까지 우리는 **"학습이 끝나면 요약 한 줄을 차곡차곡 쌓는"** 데까지 만들었습니다.
Phase 5는 그렇게 쌓인 기록을 **사용자가 볼 수 있는 그래프와 리포트로 펼쳐 보여주는** 마지막 단계입니다. 통장에 입금만 하던 것을, 이제 **잔액과 추이 그래프가 있는 가계부**로 보여주는 셈입니다.

- **📊 통계 히스토리(`/stats`)**: 내 학습을 한 화면에 그래프로. ①학습한·졸업한 단어 수, ②**학습 캘린더(잔디 심기 히트맵)**, ③**14일 학습량·점수 추이 그래프**, ④**카테고리별 숙련도 막대**.
- **📅 주간 리포트(`/report`)**: 최근 7일 성적표 — "7일 중 며칠 했는지", 총 학습 단어, 평균 점수·별점, **전주 대비 얼마나 늘었는지**, 집중한 분야, 연속 학습일.
- **🔔 주간 리포트 자동 알림**: 새로운 한 주가 시작되고 처음 들어오면, 대시보드에서 리포트가 **아래에서 스르륵 올라옵니다**(주 1회, 닫기 가능).
- **✅ 최종 QA**: 새 화면들을 **밝은 모드/어두운 모드** 양쪽에서 점검하고, 눈이 불편한 분(스크린리더)도 쓸 수 있게 라벨을 붙이고, 모바일 화면 크기에서 깨지지 않는지 확인.

> 핵심 한 줄: **"쌓인 기록"이 마침내 "눈에 보이는 성장 곡선"이 되었다.**

## 2. 가장 중요한 설계 — "계산은 한곳에서, 화면은 그걸 그리기만"

그래프를 그리는 코드와 **숫자를 계산하는 코드**를 일부러 갈라놓았습니다.

- **계산 담당**(`lib/stats/aggregate.ts`): "최근 70일 중 어느 날 몇 단어 했나", "14일 추이", "이번 주 vs 지난주" 같은 **순수 계산만** 하는 부품. 화면이나 인터넷과 전혀 얽혀 있지 않아서, **자동 테스트로 정확성을 못 박을 수 있습니다.**
- **그리기 담당**(통계/리포트 화면): 위 계산 결과를 받아 **막대·선·잔디·막대그래프로 그리기만** 합니다.

이렇게 나눈 덕분에 "평균 점수가 잘못 나온다" 같은 문제를 화면을 켜보지 않고도 **계산 테스트 7개**(`scripts/test-stats.ts`)로 즉시 잡아낼 수 있습니다. 실제로 전부 통과합니다.

> 비유: **요리(계산)와 플레이팅(화면)을 분리**한 주방. 맛(숫자)이 틀리면 플레이팅을 보지 않고도 레시피만 검사하면 됩니다.

## 3. 화면 뒤에서 벌어지는 일 — 단계별 원리

### (5-1) 통계 히스토리 — 내 학습을 그래프로

- **요약 카드**: 지금까지 **학습한 단어**와 **졸업한 단어**(복습 졸업 규칙을 두 번 통과한 단어) 개수.
- **학습 캘린더(히트맵)**: GitHub 잔디처럼 **최근 10주를 작은 칸**으로. 그날 많이 했으면 **진한 보라**, 적게 했으면 **연한 보라**, 안 했으면 회색. 한눈에 "꾸준함"이 보입니다.
- **학습량·점수 추이 그래프**: 최근 14일을 **보라 막대(그날 학습 단어 수)** + **노란 선(그날 평균 점수)** 으로 겹쳐 그립니다. 양이 늘면서 점수도 오르는지 한 화면에서 확인.
- **카테고리별 숙련도**: 12개 주제(인사·여행·직장…)별로 **졸업/학습 비율**을 막대로. "여행은 80% 외웠고 IT는 30%구나"가 보입니다.
- 기록이 하나도 없으면? 빈 화면 대신 *"첫 세트를 끝내면 여기에 그래프가 채워져요"* 라는 **친절한 빈 상태**.

> ⚙️ **개발 중 잡은 한 토막**: 그래프 그리는 라이브러리(recharts)가 **막대를 0에서부터 자라나게 하는 애니메이션** 때문에, 자동 점검용 브라우저에서는 막대가 0높이에 멈춰 **안 보이는** 문제가 있었습니다. → 막대·선의 **자라나기 애니메이션을 끄니** 즉시 또렷하게 나왔습니다(학습 흐름과 무관한 장식이라 끄는 게 오히려 안정적).

### (5-2) 주간 리포트 — 한 주 성적표

- 최근 7일을 모아 **"7일 중 N일 학습 / 총 단어 / 평균 점수 / 평균 별점"** 을 보여주고, **지난주와 비교해 얼마나 늘었는지**(▲+2일, ▲+20단어)를 초록 화살표로 표시합니다.
- **집중한 분야**(이번 주 가장 많이 본 주제)와 **가장 잘 외운 분야**(졸업 비율 1위)를 카드로 칭찬.
- 빨강·꾸중 톤은 일절 없이 *"이 페이스 좋아요!"* 같은 **항상 긍정** 메시지(우리 앱의 철칙).

### (5-2) 주간 리포트 자동 노출 — "새 주가 왔어요"

- 브라우저에 **"이 사람이 마지막으로 리포트를 본 주(週)"** 를 적어 둡니다(`tv:report:lastSeenWeek`).
- 새로운 한 주가 시작되고 대시보드에 처음 들어오면, 적어둔 주와 지금 주가 다르므로 **리포트가 자동으로 올라옵니다.** 닫으면 "이번 주는 봤음"으로 도장 → 같은 주에는 다시 안 뜹니다. 언제든 [주간 리포트] 메뉴로 다시 볼 수 있습니다.

### (5-3) 최종 QA — 마무리 점검

- **밝은 모드/어두운 모드** 양쪽에서 잔디·막대·선·글자 대비가 충분한지 실제로 보고 확인.
- **스크린리더 사용자**를 위해 잔디 영역엔 "최근 10주 동안 N일 학습", 숙련도 막대엔 "○○ 숙련도 N%" 같은 **음성 설명 라벨**을 붙였습니다(색뿐 아니라 숫자로도 전달).
- 모바일(가로 390px)에서 가로 스크롤 없이 잘 들어가는지, 동작을 줄이는 설정(`prefers-reduced-motion`)을 켠 사람에겐 과한 모션이 없는지 확인.

## 4. 로그인 vs 게스트 — 통계는 어디서 오나

| 상황 | 통계 데이터 출처 |
|---|---|
| **로그인 + 온라인** | 클라우드(Supabase)에서 **전체 이력**(세션 요약 + 단어별 진도)을 불러와 모든 그래프가 채워짐. 다른 기기에서도 동일. |
| **게스트(로그인 안 함)** | 세션 요약은 기기(LocalStorage)에 누적 → **캘린더·추이·주간 리포트는 동작**. 단, 단어별 진도는 메모리 기반이라 **새로고침하면 초기화**되어 "졸업 단어 수·카테고리 숙련도"는 비어 보일 수 있음(로그인하면 해소). |

> 어느 경우에도 통계 화면이 **에러로 멈추지 않고**, 데이터가 없으면 친절한 빈 상태로 안내합니다.

## 5. 만든 파일·바뀐 파일 한눈에

**새로 만든 것**
- `lib/stats/aggregate.ts` — 모든 통계 **계산 엔진**(캘린더·14일 추이·합계·카테고리 숙련도·주간 리포트·주 시작일).
- `lib/stats/categoryMap.ts` — 단어 번호 → 카테고리 빠른 대응표.
- `lib/sync/localStats.ts` — 게스트용 세션 요약 **로컬 기록함**.
- `hooks/useStatsData.ts` — 로그인이면 클라우드, 게스트면 로컬에서 통계 원천을 불러오는 **데이터 공급기**(로그인 여부에 맞춰 자동 분기 + 첫 진입 시 데이터 적재 보장).
- `components/stats/*` — `StatsView`(전체 배치) · `StudyCalendar`(잔디 히트맵) · `TrendChart`(막대+선 그래프) · `CategoryMasteryList`(숙련도 막대).
- `components/report/*` — `WeeklyReportCard`(리포트 카드 본체) · `WeeklyReportView`(리포트 페이지) · `WeeklyReportAuto`(새 주 자동 시트).
- `scripts/test-stats.ts` — 통계 계산 자동 검증(7/7 통과).

**손본 것**
- `app/stats/page.tsx` · `app/report/page.tsx` — "준비 중" 자리표시를 **실제 화면**으로 교체.
- `stores/userStore.ts` — 세션 결산 시, 게스트면 로컬 기록함에도 요약 한 줄을 남기도록 보강.
- `lib/sync/userData.ts` — 통계용 **전체 이력 불러오기**(`loadStatsData`) 추가.
- `components/dashboard/Dashboard.tsx` — 새 주 자동 리포트 시트(`WeeklyReportAuto`) 탑재.
- 차트 라이브러리 **recharts** 추가.

## 6. 마스터 브리핑 (사용자 여정으로)

> 한 주 동안 꾸준히 학습한 사용자가 월요일 아침 앱을 엽니다. 대시보드 아래에서 **이번 주 리포트가 스르륵 올라오며** *"7일 중 6일 학습, 지난주보다 +2일·+20단어, 평균 78점, 연속 5일째 — 이 페이스 좋아요!"* 라고 칭찬합니다. 사용자가 [통계 히스토리]를 누르면, **잔디처럼 칠해진 학습 캘린더**로 자기 꾸준함을 한눈에 보고, **보라 막대와 노란 선 그래프**로 "양도 늘고 점수도 오르는" 흐름을 확인하고, **카테고리 막대**로 "여행은 거의 졸업, IT는 더 해야겠다"를 깨닫습니다. 이 모든 숫자는 화면 뒤의 **계산 엔진**이 클라우드(또는 게스트면 기기)에 쌓인 세션 기록을 읽어 즉석에서 빚어낸 것입니다. 밝은 모드든 어두운 모드든, 작은 휴대폰 화면이든, 음성으로 읽는 분이든 똑같이 또렷하게 전달됩니다.
>
> **결과: Phase 0~4가 만든 "학습 루프"가, Phase 5에서 마침내 "보이는 성장과 동기부여"로 완성되었습니다. MVP 전 구간(0→5)이 하나의 여정으로 이어집니다.**
