# Phase 7 개발 노트 — 진단·채점 신뢰도 (믿을 수 있는 레벨·채점)

> 비개발자(기획자)도 "사용자가 무엇을 보고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 7에서 무엇을 했나

Phase 0~6으로 **기능과 콘텐츠**는 다 갖춰졌습니다. Phase 7은 학습자가 가장 쉽게 **"이 앱 좀 미덥지 않은데?"** 하고 떠나게 만드는 두 지점을 손봤습니다.

1. **레벨이 운으로 정해지는 문제** — 레벨 테스트가 매번 *랜덤 문제*라, 운 좋게 쉬운 문제가 나오면 레벨이 높게, 운 나쁘면 낮게 잡혔습니다.
2. **맞는 답인데 틀렸다고 나오는 문제** — `I'll`이라고 썼는데 정답이 `I will`이라 오답 처리되면, 학습자는 "내가 맞았는데?" 하고 의욕이 꺾입니다.

이 단계의 한 줄: **"진단(레벨)과 판정(채점)을 운·트집이 아니라 신뢰할 수 있게 만들었다."**

- **🎯 앵커 레벨테스트**: 매번 똑같은 **검증된 대표 문항 12개**로 진단 → 운 요소 제거.
- **⚖️ 다요소 채점**: 정답 개수만 보지 않고 **힌트를 얼마나 썼는지·얼마나 헤맸는지·얼마나 느렸는지**까지 반영해 *추천 시작 레벨*을 산출.
- **🔧 임시 레벨 자동 보정**: 테스트 결과는 "확정"이 아니라 **"추천"**. 실제 학습 3세트(30문항) 동안 자동으로 딱 맞게 조정.
- **✅ 정답 허용범위 확대**: `I'll`=`I will`, `well-known`=`well known`, `LET'S`=`let's` 처럼 **같은 뜻은 맞다고 인정**. 단, 철자가 진짜 다르면(`recommend`≠`recommended`) 확실히 차단.
- **💡 단계별 힌트**: 틀릴 때마다 힌트가 **첫 글자 → 단어 수 → 청크(덩어리)** 로 점점 친절해짐.

## 2. 가장 중요한 변화 — "운"과 "트집"을 걷어내기

### ① 레벨 테스트: 매번 똑같은 시험지 (7-1)

예전엔 문제를 **무작위로 뽑아** 출제했습니다. 같은 사람이 두 번 보면 결과가 달라질 수 있었죠.

이제는 **시험지가 고정**입니다. 화면 뒤에서 `buildAnchorTest`라는 도구가 2,520문장 중에서 **"가장 대표적이고 자주 쓰이는" 문항을 레벨별 4개씩, 서로 다른 주제로** 자동 선별해 고정 시험지(`anchorTest.json`)를 만들어 둡니다.

> 비유: 매번 다른 문제를 즉석에서 뽑던 쪽지시험을, **검증된 표준 모의고사 한 부**로 바꾼 셈. 누가 봐도 같은 난이도라 결과를 믿을 수 있습니다.

### ② 채점: 정답 개수 + "어떻게 맞혔나" (7-2)

예전엔 **맞은 개수만** 셌습니다. 그래서 "힌트를 다 보고 세 번 만에 맞힌 사람"과 "한 번에 척척 맞힌 사람"이 똑같은 점수를 받았습니다.

이제는 한 문제를 맞혀도 **품질**을 봅니다.
- 힌트를 봤다 → 점수 깎임
- 여러 번 틀리고 맞혔다 → 점수 깎임
- 너무 오래 걸렸다(느림) → 점수 깎임
- 긴 표현을 단번에 깔끔히 맞혔다 → 살짝 보너스

그리고 **어려운 레벨의 문제를 맞힐수록 더 큰 비중**으로 칩니다. 이 모든 걸 합쳐 0~100% 비율을 내고 **추천 시작 레벨**을 정합니다.

> 결과 화면도 바뀌었습니다. "**Lv.2 확정!**"(단정적·부담)이 아니라 *"추천 시작 레벨 Lv.2 — 학습하며 자동으로 딱 맞게 조정돼요 🙂"*(격려·여지) 라고 보여줍니다.

### ③ 레벨은 "추천"일 뿐, 진짜는 학습이 정한다 (7-3)

시험 한 번으로 레벨을 못 박지 않습니다. 테스트 결과는 **임시(추천) 레벨**로 시작하고, 이후 **실제 학습 3세트(30문항)**의 첫 시도 정답률을 화면 뒤에서 조용히 모읍니다.

- 30문항이 쌓이기 전 → 계속 **임시 상태**(언제든 조정 가능)
- 30문항 도달 → 정답률로 **자동 확정**: 아주 잘하면(85%+) 한 단계 **올리고**, 많이 힘들어하면(45%↓) 한 단계 **내리고**, 적당하면 **유지**

> 비유: 입학 배치고사로 임시 반을 정한 뒤, **첫 한 달 수업 태도를 보고 반을 확정**하는 것과 같습니다. 시험 당일의 컨디션·운에 1년이 묶이지 않습니다.

### ④ "같은 말"은 맞다고 인정 (7-4)

화면 뒤에 **정답을 표준형으로 바꿔 비교하는 자**(`canonicalAnswer`)를 뒀습니다. 입력과 정답을 둘 다 같은 규칙으로 다듬은 뒤 비교합니다.

| 학습자가 쓴 것 | 정답 | 예전 | 지금 |
|---|---|---|---|
| `I'll` | `I will` | ❌ 오답 | ✅ 정답 |
| `dont` 아니라 `don't` | `do not` | ❌ 오답 | ✅ 정답 |
| `well known` | `well-known` | ❌ 오답 | ✅ 정답 |
| `LET'S`(대문자·곱슬 따옴표) | `let's` | ❌ 오답 | ✅ 정답 |
| `an apple` ↔ `a apple` | (관사 차이) | ❌ 오답 | ✅ 정답 |
| `recommended` | `recommend` | — | ❌ **여전히 오답(정상)** |

> 핵심: **너그럽되 헐겁지 않게.** 축약·하이픈·대소문자처럼 "같은 말"은 통과시키되, 철자가 진짜 다른 단어(시제·파생어)는 학습 가치를 위해 **확실히 막습니다.** 특정 단어에 별도 정답을 더 인정하고 싶으면 `accepted_answers`에 적어두면 됩니다.

### ⑤ 틀릴수록 친절해지는 힌트 (7-5)

오답일 때 힌트가 단계적으로 더 도와줍니다. 특히 `make small talk` 같은 **여러 단어 표현**에서 빛납니다.

- **1번째 틀림** → 첫 글자: `M`
- **2번째 틀림** → 단어 수 + 머리글자: `3개 단어 · M S T`
- **3번째 틀림** → 청크(덩어리) 공개: `make ____ ____`

> 한 글자도 모를 때 무작정 정답을 깔지 않고, **"스스로 떠올릴 만큼만"** 점점 열어줘서 학습 효과를 지킵니다.

## 3. 자동 검사기 — 두뇌(로직)는 화면 없이도 검증한다

레벨 채점·보정·정답인정·힌트는 모두 **화면과 분리된 순수 계산 부품**으로 만들었습니다. 그래서 브라우저를 켜지 않고도 **자동 테스트 23개**(`npm run test:phase7`)로 정확성을 못 박습니다 — 전부 통과합니다.

- 채점(5): 깔끔히 다 맞힘→Lv.3 / 다 틀림→Lv.1 / **같은 정답수라도 힌트·저속이면 점수↓** 등
- 보정(6): 30문항 전엔 임시 유지 / 30 도달 시 고·저정답률 조정·유지
- 정답인정(8): 축약·하이픈·관사·대체정답 통과, **핵심 오답 차단**
- 힌트(4): 단계별 첫 글자→단어 수→청크 공개

> 비유: 자동차를 도로에 내보내기 전 **엔진을 따로 떼어 계기판으로 검사**하는 것. 눈으로 못 보는 오작동을 숫자로 잡습니다.

## 4. 화면 뒤 데이터 — 클라우드에 한 칸 더 (그리고 무중단 장치)

레벨 자동 보정을 기억하려면 클라우드(서버 DB)에 **칸 3개**가 더 필요합니다: "임시 레벨인가? / 지금까지 보정에 쓴 문항 수 / 그중 맞힌 수". 이를 위한 **마이그레이션(칸 추가) SQL**을 준비해뒀습니다(`v7_user_state_calibration.sql`).

- ⚠️ **운영 반영 1회**: 이 SQL을 Supabase 관리 화면(SQL Editor)에 한 번 붙여넣어 실행해야 보정값이 클라우드에 저장됩니다(Phase 6 때와 같은 방식).
- 🛟 **안전장치(무중단)**: 혹시 이 칸이 아직 없어도 **앱이 멈추지 않습니다.** 저장이 거부되면 자동으로 *기존 칸(연속학습일·레벨 등)만 다시 저장*하도록 했습니다. 즉 마이그레이션을 늦게 해도 학습·기록은 끊기지 않고, 보정값만 적용 후부터 쌓입니다.
- 적용됐는지 확인: `node --env-file=.env.local --experimental-strip-types scripts/checkUserStateColumns.ts`

> 비유: 가계부에 새 항목 칸을 추가하는 작업. 칸을 아직 안 만들었어도 **기존 가계부 기록은 정상**이고, 칸을 만든 날부터 새 항목이 채워집니다.

## 6. 마스터 브리핑 (한 문단 요약)

> Phase 7은 학습자가 떠나게 만드는 두 불신 — **"레벨이 운으로 정해진다"**, **"맞는 답인데 틀렸다고 한다"** — 을 걷어낸 단계입니다. 레벨 테스트를 매번 같은 **검증된 대표 문항 12개**로 고정(7-1)하고, 채점을 정답 개수만이 아니라 **힌트·재시도·속도·멀티워드까지 반영한 가중 점수**로 바꿔 **"추천 시작 레벨"**(7-2)을 제시합니다. 그 추천조차 못 박지 않고, 실제 학습 **3세트(30문항)** 동안 정답률로 **자동 확정·조정**(7-3)합니다. 또 `I'll`=`I will`, `well-known`=`well known` 같은 **"같은 말"은 정답으로 인정**하되 철자가 진짜 다른 건 막고(7-4), 틀릴 때마다 **첫 글자→단어 수→청크**로 점점 친절해지는 힌트(7-5)를 줍니다. 이 모든 두뇌 로직은 화면과 분리해 **자동 테스트 23개로 검증**(전부 통과)했고, 클라우드 저장은 **마이그레이션 전에도 멈추지 않는 무중단 장치**를 넣었습니다.
>
> **결과: 진단과 채점이 "운·트집"에서 "신뢰"로 바뀌었습니다. 다음은 Phase 8(장기 기억·적응)로, 한 세트가 아닌 누적 안정성 기반의 복습·적응 출제를 만들 차례입니다.**
