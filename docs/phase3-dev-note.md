# Phase 3 개발 노트 — 섀도잉 챌린지 · 발음 채점 고도화

> 비개발자(기획자)도 "사용자가 무엇을 보고, 무엇을 누르고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 3에서 무엇을 만들었나

Phase 1·2가 **"눈과 손"**(읽고 타이핑하는 학습)이었다면, Phase 3는 여기에 **"귀와 입"**을 붙였습니다.

- **🔊 듣기(TTS)**: 타이핑 정답을 맞히면 컴퓨터가 그 문장을 **소리 내어 읽어 줍니다.** 느리게 다시 듣기도 됩니다.
- **🎤 따라 말하기(섀도잉)**: 들은 문장을 마이크에 대고 따라 말하면, 얼마나 비슷한지 **별점(⭐)** 으로 돌려줍니다.
- **🧮 똑똑한 채점**: 발음을 100점 만점으로 채점하되, 컴퓨터가 잘못 알아들어도(예: see를 sea로) **억울하게 0점이 되지 않게** 여러 방법으로 비교합니다.
- **🛟 어떤 환경에서도 안 멈춤(3단계 폴백)**: 마이크가 안 되는 브라우저·오프라인·아이폰 홈화면 앱이어도 학습이 **절대 끊기지 않습니다.**

> 핵심 한 줄: **발음 단계는 "시험"이 아니라 "도전"이다.** 실패해도 유저 탓이 아니고, 언제든 [건너뛰기]로 넘어갈 수 있다.

---

## 2. 가장 중요한 원칙 — "평가가 아니라 도전"

이 Phase에서 우리가 가장 신경 쓴 것은 **사용자가 기죽지 않게** 만드는 것입니다.

| 보통 앱이 하는 실수 | 우리가 한 선택 |
|---|---|
| "틀렸습니다" 빨간 X | "한 번 더 가볼까요?" 부드러운 격려 (빨강·경고톤 금지) |
| 마이크가 안 들리면 0점 | "🎤 잘 안 들렸어요" → **점수 안 매기고 다시 시도** (실패 아님) |
| 숫자 점수로만 줄 세우기 | **별점(⭐⭐⭐)** 으로 기분 좋게 |
| 발음 안 되면 진행 막힘 | 언제나 **[건너뛰기]** 버튼 제공 — 타이핑이 진짜 핵심, 발음은 보너스 |

> 쉽게: **마이크가 말썽이어도 "내 잘못"처럼 느껴지지 않게.** 이게 리텐션(다시 오게 만들기)의 핵심입니다.

---

## 3. 3단계 폴백 — "기기 사정에 맞춰 알아서 변신"

브라우저마다 마이크·소리 기능 지원이 다릅니다. 그래서 앱이 **켜질 때 기기 사정을 살펴보고** 셋 중 하나로 자동 변신합니다. (담당: `useSpeechSupport`라는 "환경 감지기")

| 기기 사정 | 변신 모드 | 사용자 경험 |
|---|---|---|
| 소리 ✅ + 마이크 ✅ (Chrome/Edge) | **풀 모드** | 듣고 → 마이크로 따라 말하기 → **별점 채점** |
| 소리 ✅ + 마이크 ❌ (일부 Firefox 등) | **리스닝 모드** | 듣고 따라 말한 뒤 **본인이 직접 [잘 했어요/다시]** 체크 |
| 둘 다 ❌ / 오프라인 / 아이폰 홈화면 앱 | **타이핑 전용** | 발음 단계를 **조용히 건너뛰고** 다음 문제로 |

- 마이크가 안 되는 환경에서는 화면 위에 **닫을 수 있는 작은 안내 띠**가 뜹니다: *"🔊 발음 연습은 Chrome/Edge에서 가장 잘 동작해요"* — 학습을 막지 않습니다.
- 마이크는 **인터넷이 있어야** 동작합니다(목소리를 잠깐 외부 음성 서버로 보내 글자로 바꿉니다). 그래서 **오프라인이면 자동으로 타이핑 전용** 으로 내려갑니다.

> 쉽게: **"이 폰에서 되는 만큼만, 안 되면 깔끔하게 생략."** 사용자는 자기 기기가 뭘 못하는지 신경 쓸 필요가 없습니다.

---

## 4. 화면 뒤에서 벌어지는 일 — 한 문항의 흐름

타이핑 정답을 맞힌 순간부터 따라가 봅시다. (담당 화면: `ShadowingView`)

1. **자동 낭독 (3-2)**: 정답 직후 컴퓨터가 문장을 읽어 줍니다. 이때 동그란 스피커 아이콘이 **물결치듯 펄스** 합니다(지금 소리 나는 중이라는 신호). → [느리게 다시 듣기]를 누르면 0.7배속으로 또박또박.
   - *세심한 처리*: 사용자가 **다른 탭으로 가면 소리를 잠깐 멈췄다가, 돌아오면 이어서** 재생합니다(브라우저가 백그라운드에서 소리를 끊어버리는 문제 대응).
2. **따라 말하기 (3-3)**: 하단 큰 [따라 말하기 🎤] 버튼을 누르면 마이크가 켜지고 "듣고 있어요…" 표시가 뜹니다. 말이 끝나면 컴퓨터가 내 말을 **글자로 받아 적습니다.**
3. **채점 & 별점 (3-3+3-5)**: 받아 적은 글자를 정답 문장과 비교해 0~100점을 매기고, 별점으로 변환합니다.
   - ⭐⭐⭐ (85점+) → "Perfect Echo!" + **컨페티(축하 색종이)**
   - ⭐⭐ (60~84점) → "Nice!"
   - ⭐ (60점 미만, 그래도 인식은 됨) → "Keep going" + **틀린 단어에 붉은 물결 밑줄**
   - 🎤 (아예 못 알아들음) → "잘 안 들렸어요, 한 번 더?" — **점수 없음, 무한 재시도**
4. **다음으로 (3-4)**: 별점을 본 뒤 [다음 문제]를 누르거나, **3초 뒤 자동으로** 다음 문항으로 부드럽게 넘어갑니다.

> 별점은 **톡톡 튀어나오는 모션**으로 등장하고, 색깔만이 아니라 **개수와 글자("Nice!")로도** 표시합니다(눈이 불편한 분도 알 수 있게 — 접근성).

---

## 5. "똑똑한 채점"은 어떻게 억울함을 줄이나 (3-5의 핵심)

마이크(음성 인식)는 자주 잘못 알아듣습니다. 단순히 "글자가 똑같나?"로만 채점하면 **억울한 0점**이 쏟아집니다. 그래서 우리는 한 단어를 **네 가지 잣대**로 비교하고, 그중 **가장 후한 점수**를 줍니다. (담당: `scoreShadowing`)

1. **완전히 같은 단어?** → 100% 인정.
2. **소리가 같은 단어(동음이의어)?** → `see`↔`sea`, `to`↔`too`, `their`↔`there` 같은 흔한 헷갈림은 **거의 만점**으로 인정. (사람 귀에는 똑같으니까)
3. **철자가 비슷한가?** → 한두 글자 틀린 정도면 비슷한 만큼 점수. (예: `tikit` vs `ticket`)
4. **소리 나는 느낌이 비슷한가(발음 코드)?** → 철자는 달라도 발음 패턴이 닮았으면 부분 점수. (간이 발음 코드 방식)

그리고 채점 전에 **말투를 표준화**합니다: 대소문자·구두점을 없애고, **줄임말을 풀어 줍니다**(`don't`→`do not`, `I'm`→`i am`, `gonna`→`going to` 등 30여 종). 덕분에 줄여 말하든 풀어 말하든 똑같이 인정됩니다.

마지막으로:
- **빠뜨리거나 약하게 인식된 단어**는 `weakWords`로 모아 **문장에서 붉게** 보여 줍니다 → "여기만 다시 말해볼까?"
- **쓸데없이 더 말한 단어**는 아주 조금만 감점.
- 마이크 신뢰도가 너무 낮으면 점수를 매기지 않고 "다시 한 번"으로 보냅니다. (신뢰도로 점수를 **부풀리지 않음** — 공정성)

> 쉽게: **"사람 채점관처럼 후하게, 하지만 약점은 콕 집어서."**

### 이 채점이 진짜 맞는지 — 자동 검증

코드가 의도대로 동작하는지 **자동 테스트 6개**로 못 박아 두었습니다(`scripts/test-score.ts`). 모두 통과합니다:

| 상황 | 기대 결과 | 통과 |
|---|---|---|
| 똑같이 말함 | 100점 / ⭐⭐⭐ | ✅ |
| 줄임말로 말함 (`don't`) | 고득점 | ✅ |
| 동음 오인식 (to/too, see/sea) | **0점 아님, 고득점** | ✅ |
| 절반만 말함 | 중간 점수 + 빠진 단어 표시 | ✅ |
| 아무 말도 안 들림 | "다시 한 번"(점수 없음) | ✅ |
| 신뢰도 너무 낮음 | "다시 한 번" | ✅ |

## 6. 만든 파일·바뀐 파일 한눈에

**새로 만든 것**
- `hooks/useSpeechSupport.ts` — 기기 환경 감지기(풀/리스닝/타이핑전용 결정).
- `types/speech.d.ts` — 음성 인식 기능의 타입 정의(코드가 헷갈리지 않게).
- `lib/speech/tts.ts` — 문장 읽어주기(+느리게, +탭 전환 대응).
- `lib/speech/stt.ts` — 마이크로 받아쓰기(+오류 종류 구분).
- `lib/shadowing/score.ts` — 똑똑한 발음 채점 엔진.
- `components/study/ShadowingView.tsx` — 따라 말하기 화면(3모드 통합).
- `components/study/StarRating.tsx` — 톡톡 튀는 별점 표시.
- `components/study/UnsupportedBanner.tsx` — 미지원 환경 비차단 안내 띠.
- `scripts/test-score.ts` — 채점 자동 검증 테스트.

**손본 것**
- `components/study/QuestionView.tsx` — 정답 후 자리표시(placeholder) 대신 진짜 섀도잉 화면 연결.
- `components/study/StudySession.tsx` — 환경 감지 + 안내 띠 + 모드 전달.
- `components/study/SessionResult.tsx` — 결과창에 **평균 별점**과 단어별 ⭐ 표시 추가.
- `stores/sessionStore.ts` / `types/index.ts` — 문항별 별점·점수·건너뜀 여부 기록(다음 Phase 통계·복습의 재료).

## 7. 다음 Phase에 넘기는 재료

이번에 문항마다 **별점·점수·건너뜀 여부**를 차곡차곡 기록해 두었습니다. 이건 **Phase 4**에서:
- 세션 결과를 클라우드에 저장하고,
- "별 1개 이하 → 복습 대상" 같은 **복습 졸업 규칙**을 돌리고,
- **Phase 5**의 통계·주간 리포트 그래프를 그리는 **연료**가 됩니다.

> 한 줄 요약: **Phase 3에서 앱이 "듣고 말하게" 되었고, 그 결과를 다음 단계가 쓸 수 있게 기록까지 남겼다.**

## 마스터 브리핑 (사용자 여정으로)

> 사용자가 "ticket"을 타이핑으로 맞혔습니다. **화면이 초록으로 번쩍이고**, 곧바로 컴퓨터가
> *"I need a ticket to Busan."* 을 또박또박 읽어 줍니다(스피커가 물결칩니다). 사용자는 [느리게 다시 듣기]로
> 한 번 더 듣고, 큰 [따라 말하기 🎤] 버튼을 눌러 문장을 따라 말합니다. 잠시 후 **별 세 개가 톡톡 튀어나오며
> "Perfect Echo!" 와 색종이가 쏟아집니다.** 만약 마이크가 잘 안 들렸다면? 빨간 X 대신 *"🎤 한 번 더 해볼까요?"*
> 가 뜨고, 부담 없이 다시 시도하거나 [건너뛰기]로 넘어갈 수 있습니다. 마이크가 아예 없는 브라우저라면?
> 사용자는 아무것도 막힌 줄 모른 채 타이핑만으로 학습을 이어갑니다.
>
> **결과: "외워서 쓰고(타이핑) → 듣고 따라 말하는(섀도잉)" 회화 학습 루프가 손끝과 입으로 완성되었습니다.**
