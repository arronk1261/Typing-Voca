# Phase 10 개발 노트 — 동기부여 시스템 (애플 활동 스타일)

> 비개발자(기획자)용 브리핑. "사용자가 무엇을 보고, 어떤 동작을 하고, 화면이 어떻게 반응하는지" 관점.

## 한 줄 요약

매일 돌아올 이유(닫을 **학습 링**), 모을 이유(**배지 컬렉션·경험치**), 놓치지 않을 안전장치(**스트릭 동결권**)를 학습 루프 위에 얹었습니다. 애플 활동 앱의 4가지 원리 — ①채워야 할 고리 ②과거의 나와 경쟁 ③달성 순간의 즉각 축하 ④수집욕·희소성 — 을 그대로 가져왔습니다.

## 사용자가 보게 되는 것

1. **대시보드 상단의 "오늘의 링" 3개**
   - 🟣 학습 / 🟠 복습 / ⭐ 발음 — 애플 활동 링처럼 동심원으로 채워집니다.
   - 목표치는 **과거의 내 평균을 기준으로 자동 조정**(최소 하루 10단어). 무리한 목표로 좌절하지 않게 바닥값·상한을 뒀습니다.
   - 세 링을 다 채우면 축포(confetti)가 터집니다(하루 한 번).
   - 헤더에는 누적 **경험치(XP)** 와 보유 **동결권(🧊)** 이 함께 보입니다.

2. **세트 완료 화면의 보상**
   - 점수와 함께 **+XP** 가 표시됩니다.
   - 이번 세트로 새 배지를 땄다면, 배지가 **톡 튀어나오는 시트**가 confetti·진동과 함께 올라옵니다. "확인" 한 번이면 바로 학습으로 돌아갑니다(흐름 방해 0).
   - 신기록을 세우면("최장 연속!", "역대 최고 정확도!") 격려 토스트가 뜹니다.
   - 동결권을 새로 얻으면 "하루 쉬어도 연속이 지켜져요" 안내가 뜹니다.

3. **배지 컬렉션 화면**(대시보드 → 배지)
   - 획득한 배지는 컬러로, 아직 못 딴 배지는 **흐릿한 실루엣 + 진행바("앞으로 N")** 로 보여 수집욕을 자극합니다.
   - 연속 / 누적 / 기량 / 이달의 챌린지 / 탐험 5개 묶음으로 정리됩니다.
   - 상단에 획득 수·경험치·동결권 요약.

4. **주간 리포트의 "이번 주 미션"**
   - 한 주 동안 단어 목표(50개) 진행바. 달성하면 그 달의 시즌 배지를 받습니다.

## 배지 종류 (총 26종)

- **연속 🔥**: 3·7·14·30·60·100·365일, 주말 전사, 불사조(끊겼다 다시 7일)
- **누적 📚**: 학습 50·100·250·500·1000단어, 졸업 10·50·100개, 카테고리 마스터
- **기량 ⭐**: 퍼펙트 세트, 골든 보이스, 스피드러너, 오답 정복
- **이달의 챌린지 📅**: 매월 15일 학습(달마다 새로 도전)
- **탐험 🧭**: 5개 카테고리 경험, 올빼미(심야), 아침형

## 안전장치 — 스트릭 동결권

7일 연속에 도달할 때마다 동결권 1개를 얻습니다(최대 3개). 하루(딱 하루)를 빠져도 동결권이 있으면 연속이 끊기지 않습니다. **"한 번 끊기면 포기"하는 이탈을 막는** 가장 강력한 장치입니다.

## 지켜진 원칙 (claude.md 규칙 6 / T2 / T3)

- **긍정 강화만**: 놓친 배지·끊긴 연속을 빨강·실패·벌점으로 표시하지 않습니다.
- **학습 무중단**: 배지 판정·동기화는 전부 best-effort. 배지 테이블이 아직 없거나(마이그레이션 전) 저장이 실패해도 학습은 0초도 멈추지 않습니다.
- **새 점수 로직 없음**: 기존 학습 데이터(progress·study_sessions·SM-2 신호)를 *읽어서* 판정만 합니다.
- **흐름 방해 0 / 접근성**: 획득 연출은 300ms대, `prefers-reduced-motion` 존중, 색+아이콘+`aria-live` 이중 표시, 48px 터치 타깃.

## 기술 메모 (개발자용)

### 파일 구성
- `src/lib/achievements/catalog.ts` — 배지 정의(순수, 단일 소스). 각 배지는 `progress(ctx) → {current, target}` 하나로 "획득 판정"과 "진행바"를 통일.
- `src/lib/achievements/engine.ts` — `evaluate(ctx)`(새로 딴 배지) / `achievementProgress(ctx)`(컬렉션). 순수·node 테스트 가능(words.json 비의존).
- `src/lib/achievements/snapshot.ts` — 세션/현재 상태로 판정 컨텍스트 빌드(클라이언트, `wordCategoryMap` 사용).
- `src/lib/achievements/rings.ts` — 링 목표 계산(중앙값×1.05, 바닥/상한). 순수.
- `src/lib/sync/dailyRing.ts` / `studyDays.ts` — 오늘의 링 달성치·학습일 원장(로컬, 오프라인 동작).
- `src/components/dashboard/LearningRings.tsx` · `achievements/AchievementSheet.tsx` · `achievements/AchievementCollection.tsx`(+ `app/achievements/page.tsx`).
- 시즌 배지는 키를 월별로 분기(`monthly_days_2026-06`)해 매달 새로 도전.

### 데이터 (v12 마이그레이션)
- `achievements`(공개 읽기 전용 카탈로그) / `user_achievements`(본인 RLS) / `daily_rings`(본인 RLS) 신설.
- `user_state` 확장: `streak_freezes`·`xp`·`best_streak`.
- 오프라인 큐에 `user_achievements`·`daily_rings` 추가. 컬럼/테이블 부재 시 떼고 재시도 또는 폐기(무중단).
- 카탈로그 시드: `npm run seed:achievements`(Service Role 전용). 운영 반영: SQL Editor에서 `supabase/migrations/v12_achievements.sql` 1회 적용 + 시드.

### 동기부여 로직 연결
- `userStore.commitSession`: 스트릭은 `applyStreak`(동결권 적용)로 갱신 → 베스트 스트릭·동결권 적립/소모 계산 → 링 누적·학습일 기록 → `buildSnapshot`+`evaluate`로 새 배지 산출 → XP 합산·개인 기록 경신 판정. 세션 종료 시 변경분만 배치 upsert(키 입력마다 X).
