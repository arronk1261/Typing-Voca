# Phase 2 개발 노트 — 대시보드 · 학습 데이터 · Streak · 카테고리 선택

> 비개발자(기획자)도 "사용자가 무엇을 보고, 무엇을 누르고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 2에서 무엇을 만들었나

Phase 1은 **"문제 하나를 푸는 경험"** 이었다면, Phase 2는 그 위에 **"집(앱)을 만든 것"** 입니다.

- **🏠 대시보드(메인 로비)**: 앱을 켜면 가장 먼저 보이는 집의 거실. 내 레벨·누적 단어·연속 학습일(Streak)을 보여주고, "오늘의 10단어 / 카테고리 학습 / 오답 노트" 같은 문으로 안내합니다.
- **🧠 유저 데이터 기억 장치**: 학습 기록(레벨·연속일·누적 단어·선호 주제)을 **다음에 와도, 다른 기기에서도** 기억합니다. (로그인하면 클라우드에, 비로그인이면 기기 안에)
- **📝 레벨 테스트(첫 방문 온보딩)**: 처음 온 사람에게 5문제를 풀게 해 시작 레벨(Lv.1~3)을 정해 줍니다.
- **🔥 연속 학습일(Streak)**: 매일 학습하면 숫자가 올라가고 불꽃이 점점 커집니다. 하루 빠지면 리셋.
- **🗂️ 카테고리 선택 학습**: 12개 주제(여행·쇼핑·IT 등) 중 골라서, 그 주제 문장만 학습할 수 있습니다.

> 핵심 한 줄: **"한 문제 풀기"에서 → "내 학습이 쌓이고 이어지는 앱"으로 자랐다.**

## 2. 이번 Phase의 큰 그림 — '세션 안'과 '세션 밖'의 데이터

이 Phase에서 가장 중요한 설계 개념은 **데이터를 두 종류로 나눈 것**입니다.

| 구분 | 비유 | 예시 | 어디에 저장? |
|------|------|------|--------------|
| **세션 안 데이터** | 지금 풀고 있는 시험지 위의 메모 | 현재 몇 번 문제, 남은 하트, 방금 정답 여부 | 잠깐 머릿속(메모리)에만 — `sessionStore` (Phase 1) |
| **세션 밖 데이터** | 성적표/출석부처럼 오래 남는 기록 | 내 레벨, 연속 학습일, 누적 단어, 선호 주제 | 오래 보관 — `userStore` + 클라우드/기기 |

**왜 나눴나?** 글자 하나 칠 때마다 클라우드에 저장하면 느리고 데이터도 낭비됩니다.
그래서 **평소엔 머릿속(메모리)에서 빠르게 처리**하고, **"레벨 확정 / 주제 저장 / 세션 완료" 같은 중요한 길목(체크포인트)에서만** 클라우드에 한 번 기록합니다. 마치 시험을 다 본 뒤 성적을 한 번에 적는 것과 같습니다.

이 "성적표 담당 사물함"이 이번에 새로 만든 **`userStore`**(유저 데이터 보관소)입니다.

## 3. 화면 뒤에서 벌어지는 일 — 단계별 원리

### (2-1) 유저 데이터 보관소 — 기억하고, 이어지게

- **무엇을 기억하나**: 레벨, 연속 학습일, 마지막 학습 날짜, 누적 단어 수, 선호 카테고리, 그리고 단어별 학습 신호(복습 대상인지 등).
- **로그인한 사람**: 구글 계정에 묶인 클라우드(Supabase)에서 불러오고 저장합니다 → **다른 기기에서도 그대로 이어짐.**
- **로그인 안 한 사람(게스트)**: 그 기기 안 작은 저장공간(LocalStorage)에 보관합니다 → 인터넷이 없거나 로그인을 안 해도 **학습이 절대 막히지 않음**(우리 프로젝트의 절대 규칙).
- **언제 저장하나(체크포인트)**: ①레벨 테스트 끝났을 때 ②카테고리를 골랐을 때 ③(다음 Phase에서) 세션을 완주했을 때. 평소 타이핑 중엔 저장하지 않습니다.

> 쉽게: **"앱이 나를 기억한다."** 껐다 켜도, 폰을 바꿔도(로그인 시) 내 진도가 그대로.

### (2-2) 대시보드 — 집의 거실

- 앱을 켜면 보이는 첫 화면. 위쪽엔 **내 상태 카드**(현재 레벨, 누적 단어, 연속 학습일 🔥), 가운데엔 **큰 버튼들**:
  - **오늘의 10단어 학습** — 내 레벨에서 전 주제 섞어 출제
  - **카테고리 학습** — 주제 고르기 창(아래 2-5)이 열림
  - **오답 노트** — 어려웠던 단어가 있을 때만 나타남(없으면 버튼 자체가 안 보임)
- 아래쪽엔 **통계 히스토리 / 주간 리포트** 진입 버튼(내용은 Phase 5에서 채워지며, 지금은 "준비 중" 안내 화면).
- 주요 버튼은 모두 **엄지손가락이 닿는 화면 아래쪽**에 큼직하게 배치(모바일 우선 규칙).
- **로그인-퍼스트**: 클라우드가 켜져 있고 아직 로그인을 안 했다면, 거실 대신 **[Google로 계속하기]** 카드를 먼저 보여줍니다.

### (2-3) 레벨 테스트 — 첫 방문자 맞이

- 처음 온 사람(아직 레벨이 정해지지 않은 사람)에게 **5개 문제**를 타이핑으로 풀게 합니다. 문제는 쉬운 것(Lv1)부터 어려운 것(Lv3)까지 섞여 있습니다.
- **맞힌 개수로 시작 레벨 결정**: 0~1개→Lv.1, 2~3개→Lv.2, 4~5개→Lv.3.
- "이건 추정치이고 학습하면서 자동으로 조정돼요"라는 **부담 없는 안내**를 답니다(긍정 강화 톤). 모르면 [모르겠어요]로 넘어가도 됩니다.
- **한 번 보면 다시 안 봄**: 테스트를 마치면 "이 사람은 온보딩 완료"라는 표시(`onboarded`)를 저장해, 재방문 시 곧장 대시보드로 갑니다.
  - 🔧 *개발 메모*: 클라우드 표 `user_state`에 `onboarded`(참/거짓) 칸을 새로 추가했습니다. 레벨이 기본값 1로 시작하기 때문에, "레벨이 1이다"만으로는 "테스트를 봤는지"를 알 수 없어서 별도 표시가 필요했습니다.

### (2-4) 연속 학습일(Streak) — 매일 오게 만드는 장치

- **규칙**: 오늘 처음 학습을 완료하면 연속일 +1. 어제도 했으면 이어짐. **하루라도 건너뛰면 1로 리셋.** 같은 날 두 번 해도 중복으로 오르진 않습니다.
- **불꽃 비주얼**: 연속일이 길어질수록 불꽃이 커지고 빛납니다(작은 불씨 → 활활 타오르는 불꽃, 3·7·14·30일 구간). 살아있는 불꽃은 은은하게 흔들립니다(애니메이션).
- 이 계산 로직은 **날짜를 가짜로 넣어보는 테스트**로 "+1 / 이어짐 / 리셋"이 정확히 동작함을 확인했습니다.
- ⏳ *연결 시점*: "세션을 완주하면 연속일을 올린다"의 **실제 호출은 Phase 4 결과창**에서 붙입니다. 이번엔 계산기와 불꽃 그림을 먼저 완성해 둔 단계입니다.

### (2-5) 카테고리 선택 학습 — 내가 원하는 주제만

- **12개 주제**(인사·스몰토크 / 쇼핑·결제 / 음식·레스토랑 / 여행·교통 / 직장·업무 / 전화·이메일 / 감정·의견 / 약속·일정 / 건강·병원 / 일상 루틴 / 인간관계·소셜 / IT·실무)를 아이콘이 달린 **칩(작은 토글 버튼)** 으로 보여줍니다.
- **아래에서 올라오는 창(바텀시트)** 에서 여러 개를 탭으로 켜고 끕니다. 고른 개수에 따라 버튼 글자가 "2개 주제로 학습 시작"처럼 바뀝니다.
- 고른 주제는 **다음에 올 때 기본값으로 복원**됩니다(선호 주제로 저장).
- **문항이 부족하면 똑똑하게 채움(충원 규칙)**: 고른 주제의 현재 레벨 문장이 10개에 못 미치면, 같은 주제의 **인접 레벨**에서 더 가져와 10개를 맞춥니다. (단어 DB가 레벨당 300개로 넉넉해 단일 주제도 충분합니다.)
- **하나도 안 고르면** = 전체에서 출제(=오늘의 10단어와 동일).

> 브라우저 실측: 여행·IT 2개를 골라 시작 → 실제로 그 주제 문장(예: "Turn ___ at the corner")으로 10문제가 구성됨을 확인했습니다.

## 4. 이 프로젝트의 규칙을 어떻게 지켰나

- **규칙 5 (UI/UX·모바일 우선)**: 모든 화면을 390px 세로 기준으로 설계, 주요 버튼은 하단·48px 이상, 카테고리 창은 전체 모달 대신 **바텀시트**, 화면 전환엔 슬라이드/페이드 모션. 라이트·다크 모드 둘 다 점검.
- **규칙 6 (학습 무중단·데이터 경계)**: 로그인 안 해도(게스트) 모든 기능이 동작. 클라우드엔 **체크포인트에서만** 기록하고, 본인 데이터만 접근(클라우드의 RLS 보안 규칙 그대로 사용).
- **규칙 2 (주석 최소화)**: 변수·함수 이름만으로 읽히게 작성, 비자명한 한 줄에만 주석.
- **규칙 4 (비개발자 브리핑)**: 이 문서가 그 브리핑입니다.
- **긍정 강화 톤(T2)**: 레벨 테스트는 "추정치, 자동 조정", 모르면 넘어가도 괜찮다고 안내 — 평가가 아니라 출발점.

## 6. 직접 실행해 보는 법

```bash
npm run dev      # http://localhost:3000 (실제 클라우드 연동 = 로그인-퍼스트)
npm run build    # 타입·문법 검사 포함 빌드
```

- **게스트 모드로 빠르게 둘러보기(로그인 없이)**: 미리보기 설정 `dev-guest`(포트 3100)를 쓰면 클라우드 미설정 상태를 흉내 내, 로그인 없이 **레벨 테스트 → 대시보드 → 카테고리 학습**을 전부 확인할 수 있습니다. (개발/QA용)
- 첫 진입 시 레벨 테스트 5문제 → 대시보드 → [카테고리 학습]에서 주제 고르기 → 학습 시작 순으로 흐릅니다.

> ⚠️ **실제 클라우드(Supabase)를 쓰는 경우 1회 준비**: 레벨 테스트 완료 여부를 저장할 `onboarded` 칸을 추가해야 합니다. Supabase SQL Editor에서 아래 한 줄을 실행하세요(이미 있으면 무시됨).
> ```sql
> alter table public.user_state add column if not exists onboarded boolean not null default false;
> ```

## 7. 아직 가짜이거나 다음 Phase로 미룬 것

- **섀도잉(따라 말하기)**: 여전히 자리표시(placeholder). → **Phase 3**에서 음성·채점 구현.
- **연속일 실제 증가·세션 기록 저장**: 계산기와 불꽃은 완성했지만, "세션 완주 시 올리고 저장"의 실제 연결은 **Phase 4 결과창**에서.
- **복습 졸업 규칙(어떤 단어가 오답 노트에 들어가고 졸업하는지)**: **Phase 4**. 지금은 오답 노트 버튼이 "복습 대상이 있을 때만" 뜨도록 골격만 준비.
- **통계 히스토리 / 주간 리포트**: 진입 버튼만 있고 화면은 "준비 중". → **Phase 5**.
- **적응형 레벨 자동 제안**(잘하면 올려주기): **Phase 4**.

### ✅ Phase 2 마스터 브리핑 (한 문단 요약)

이제 Typing-Voca는 단순한 문제 풀이를 넘어 **하나의 앱**이 됐습니다. 사용자는 처음 오면 가벼운 **레벨 테스트**로 출발점을 잡고, **대시보드(거실)** 에서 자기 레벨·누적 단어·**연속 학습일 🔥** 을 한눈에 봅니다. 원하는 **주제(카테고리)** 만 골라 학습할 수도 있고, 이 모든 기록은 **다음에 와도, 다른 기기에서도(로그인 시)** 그대로 이어집니다. 로그인을 안 해도 학습은 멈추지 않습니다. 즉, **"매일 다시 오고 싶고, 내 진도가 쌓이는"** 학습 앱의 뼈대가 완성됐습니다.
