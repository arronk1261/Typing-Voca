# Phase 9-4 개발 노트 — SM-2 간격 알고리즘 + 리뷰 로그 (비개발자용)

복습 간격을 **고정값(1·3·7·14일)에서 단어별 적응형(SM-2)으로** 업그레이드했습니다. SM-2는 SuperMemo가 만든, 수십 년 검증된 간격 반복 알고리즘입니다.

## 9-4a — 단어마다 다른 '복습 타이밍'

- **무엇이 문제였나:** 모든 단어가 똑같이 1→3→7→14일 간격으로 돌아왔습니다. 쉬운 단어도, 자꾸 틀리는 단어도 동일하게요.
- **무엇이 바뀌나:** 이제 단어를 맞힌 **방식**에 따라 다음 복습 시점이 달라집니다.
  - **한 번에 척척(Easy)** → 다음 간격이 크게 벌어짐(자주 안 봐도 됨)
  - **첫 시도 정답(Good)** → 정상적으로 간격 증가
  - **여러 번 만에/힌트 보고(Hard)** → 간격이 짧게, 더 자주 복습
  - **하트 다 소진(Again)** → 간격 리셋, 내일 다시
- 각 단어가 "난이도계수(EF)"를 갖고, 잘 맞힐수록 올라가고 헤맬수록 내려가서 **개인별·단어별로 최적 타이밍**에 복습됩니다.

## 9-4b — 졸업 기준에 '간격' 추가

- 기존: 통과 3회면 졸업.
- 변경: **통과 3회 + 복습 간격이 14일 이상**이어야 졸업합니다.
- 효과: 자주 틀려서 간격이 잘 안 늘어나는 단어는 통과 횟수를 채워도 **더 오래 복습 더미에 남아** 확실히 익은 뒤 졸업합니다. (발음 트랙·typingOnly 졸업 등 기존 규칙은 그대로)

## 9-4c — 복습 기록 쌓기 (향후 대비)

- 매 문항의 채점 등급·경과일·난이도계수 등을 `review_logs`에 조용히 기록합니다.
- 지금 당장 화면에 보이진 않지만, **나중에 더 똑똑한 알고리즘(FSRS)으로 업그레이드할 때 학습 재료**가 됩니다.
- 기록 실패나 미적용 환경에서도 학습은 전혀 영향받지 않습니다(비핵심·폐기 처리).

## 왜 SM-2인가 (FSRS는 나중에)
- **SM-2**: 학습 데이터 없이 바로 동작, 단어별 적응, 구현·검증 단순 → **지금 단계에 최적**.
- **FSRS**(최신 SOTA): 회상 확률을 모델링하고 목표 유지율을 설정할 수 있지만, **리뷰 로그로 파라미터를 학습**해야 제값을 합니다. 그래서 9-4c로 로그를 먼저 쌓고, 데이터가 충분해지면 전환을 검토합니다.
- 상세 비교·근거는 `docs/learning-system-guide.md` §6 및 학습 이론 부록을 참고하세요.
