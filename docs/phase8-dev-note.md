# Phase 8 개발 노트 — 장기 기억·적응 (오래 기억하게, 사람마다 다르게)

> 비개발자(기획자)도 "사용자가 무엇을 보고, 화면 뒤에서 어떤 일이 일어나는지"를
> 그림처럼 이해하도록 쓴 문서입니다. 기술 용어가 나오면 바로 옆에 쉬운 비유를 붙였습니다.

---

## 1. 한눈에 보기 — Phase 8에서 무엇을 했나

Phase 0~7로 **학습 루프·콘텐츠·진단/채점 신뢰도**까지 다 갖췄습니다. Phase 8은 학습 앱의 진짜 승부처 — **"외운 걸 오래 기억하게 만들고, 사람마다 다른 속도에 맞춰주는"** 두뇌를 넣는 단계입니다.

한 줄로: **"한 세트만 보던 근시안에서, 그 사람의 학습 흐름 전체를 보는 시스템으로."**

- **🗓️ 복습 간격 장기화(8-1)**: 한 번 맞혔다고 끝이 아니라, **1일 → 3일 → 7일 → 14일**로 점점 간격을 벌리며 여러 번 만나야 졸업. (까먹기 직전에 다시 보는 게 기억에 가장 잘 남습니다)
- **📊 3요소 분리 기록(8-2)**: "이 단어를 안다/모른다" 한 덩어리가 아니라 **뜻 회상·철자·발음 세 점수로 따로** 기록. 나중에 "철자는 되는데 발음이 약한 단어"를 콕 집어줄 토대.
- **⚖️ 적응형 출제 비율(8-3)**: 신규:복습 비율을 **상황에 맞춰 자동 조정**. 초보·복귀자·오답 많은 사람마다 다르게.
- **🔗 크로스레벨 복습(8-4)**: 레벨이 올라가도 **예전 레벨에서 어려웠던 단어**를 잊지 않고 계속 복습 풀에 데려옴.
- **📈 누적 기반 승급(8-5)**: 레벨 업/다운 제안을 **한 세트의 운**이 아니라 **최근 3~5세트 누적 성적**으로 판단(흔들림 제거).
- **♻️ 졸업 후 유지 점검(8-6)**: 졸업한 단어도 **~한 달 뒤 슬쩍 한 번 더** 꺼내 "정말 장기 기억에 남았나" 확인.

---

## 2. 가장 중요한 변화 — "까먹기 직전에 다시 보기" (8-1)

사람은 **막 외운 직후**보다 **살짝 잊어갈 무렵 다시 볼 때** 가장 단단히 기억합니다(간격 반복, spaced repetition). 그래서 복습 주기를 길게 늘렸습니다.

- 예전: 두 번 통과하면 바로 졸업
- 지금: 통과할 때마다 다음 복습이 **1일 → 3일 → 7일** 뒤로 멀어지고, **기본 3번** 통과해야 졸업
- 가장 어려운 **Lv.3 관용구(idiom)**는 **4번** 통과해야 졸업 — 더 오래, 더 여러 번 곱씹게

> 비유: 벼락치기(하루에 몰아보기)가 아니라, **간격을 둔 복습 캘린더**. 잊을 만하면 한 번씩 등장해 기억을 굳힙니다.

---

## 3. "안다"를 셋으로 쪼개기 (8-2)

예전엔 한 단어를 "맞음/틀림"으로만 봤습니다. 하지만 실제론 **뜻은 아는데 철자를 틀리거나**, **철자는 되는데 발음이 약한** 경우가 많죠. 그래서 한 번의 학습을 **세 점수로 분리 저장**합니다.

| 점수 | 무엇을 보나 | 어떻게 매기나 |
|---|---|---|
| 뜻 회상 | 그 단어를 떠올렸나 | 끝내 맞히면 100, 끝까지 못 맞히면 0 |
| 철자 | 정확히 썼나 | 한 번에 100, 틀려서 다시 칠 때마다 −35 |
| 발음 | 따라 말하기 점수 | 섀도잉 STT 점수(건너뛰면 직전 점수 유지) |

> 지금은 **데이터를 쌓는 단계**입니다. 이 세 점수가 모이면 다음에 "발음 약한 단어만 모아 연습" 같은 맞춤 기능을 붙일 수 있는 **토대**가 됩니다.

## 4. 사람마다 다른 출제 (8-3 · 8-4)

### 신규:복습 비율을 상황에 맞춰 (8-3)
모두에게 똑같이 "신규 70 / 복습 30"을 주지 않습니다. 화면 뒤에서 학습자 상태를 보고 **복습 비율을 조절**합니다.

| 상황 | 복습 비율 | 왜 |
|---|---|---|
| 며칠 쉬다 돌아온 복귀자 | **70%** | 까먹은 것부터 다시 다지며 부드럽게 복귀 |
| 오답(복습 대상)이 많이 쌓임 | **60%** | 밀린 복습부터 해소 |
| Lv.1 갓 시작한 초보 | **50%** | 새것과 복습을 반반, 부담 적게 |
| 안정적으로 잘하는 중 | **30%** | 새 단어 위주로 쭉쭉 |

또 **카테고리 학습**은 실제 상황 흐름(인사 → 일상 → 약속 …) 순서로 묶어 자연스럽게 이어지게 했습니다.

### 레벨이 올라도 옛 단어를 안 버린다 (8-4)
예전엔 레벨을 올리면 **이전 레벨에서 어려웠던 단어가 복습 풀에서 사라지는** 허점이 있었습니다. 이제는 레벨과 무관하게 **"아직 복습 중인 단어"는 전부** 불러와, 레벨을 올려도 빠뜨리지 않고 계속 복습합니다.

> 비유: 3학년이 됐다고 2학년 때 약했던 부분을 안 보는 게 아니라, **약점 노트는 학년이 올라가도 계속 들고 다니는** 것.

---

## 5. 레벨 조정은 "한 판 성적"이 아니라 "최근 성적표"로 (8-5)

예전 레벨 업/다운 제안은 **바로 직전 한 세트**만 보고 결정했습니다. 컨디션 좋은 한 판에 레벨이 출렁였죠. 이제는 **최근 3~5세트(약 30~50문항) 누적**으로 판단합니다.

- **올리자고 제안**: 최근 정답률 85%↑ + 복습행(오답) 비율 15%↓ + 발음 별점 평균 2.2↑ 가 **꾸준히** 유지될 때
- **잠깐 쉬어가자고 제안**: 최근 정답률이 낮거나 복습행 비율이 높을 때
- 메시지는 늘 **격려 톤·강제 없음**: *"Lv.3 맛보기 세트 어때요?"* / *"Lv.1 기초 다지기 모드로 잠깐 쉬어가도 좋아요."*

> 비유: 하루 시험 점수가 아니라 **한 달 평균 성적표**로 반을 조정. 운 좋은/나쁜 하루에 휘둘리지 않습니다. 이 "최근 성적표"는 기기에 차곡차곡 저장됩니다.

---

## 6. 졸업했다고 끝이 아니다 (8-6)

졸업한(완전히 외운) 단어도 시간이 지나면 잊힙니다. 그래서 졸업 단어 중 **마지막으로 본 지 ~한 달 지난 것**을 가끔 한 개씩 슬쩍 세션에 끼워 넣어 **"정말 장기 기억에 남았는지"** 점검합니다.

> 비유: 졸업생에게 보내는 **가벼운 동창회 초대장**. 자주는 아니고, 잊을 만할 때 한 번.

---

## 7. 자동 검사기 — 두뇌 로직은 화면 없이 검증

이 모든 "두뇌"(간격·점수·비율·승급·유지)는 **화면과 데이터에서 분리한 순수 계산 부품**(`lib/words/adaptive.ts`, `lib/srs.ts`)으로 만들어, 브라우저 없이 **자동 테스트 25개**(`npm run test:phase8`) + **SRS 8개**(`npm run test:srs`)로 검증합니다 — 전부 통과.

- 8-1 간격/졸업(6) · 8-2 3요소(5) · 8-3 비율(5) · 8-5 승급(5) · 8-6 유지(3) · 보조(1)

> 비유: 자동차 엔진을 떼어내 **계기판으로 시험**. 눈에 안 보이는 로직 오류를 숫자로 잡습니다.

## 8. 화면 뒤 데이터 — 클라우드에 칸 3개 더 (무중단 장치 포함)

3요소 점수(8-2)를 저장하려면 클라우드에 **칸 3개**(`meaning_recall_score`·`spelling_score`·`pronunciation_score`)가 필요합니다.

- ⚠️ **운영 반영 1회**: `supabase/migrations/v8_progress_components.sql`을 Supabase SQL Editor에 한 번 실행(v6·v7과 동일).
- 🛟 **안전장치(무중단)**: 칸이 아직 없어도 **저장이 멈추지 않습니다.** 거부되면 점수 칸만 떼고 *기존 학습 기록은 그대로 저장*합니다. 적용 후부터 3요소 점수가 쌓입니다.
- 적용 확인: `npm run check:schema` (v7·v8 컬럼을 한 번에 점검)

## 10. 마스터 브리핑 (한 문단 요약)

> Phase 8은 학습 앱의 핵심 — **"오래 기억하게, 사람마다 다르게"** — 를 시스템으로 구현한 단계입니다. 복습을 **1→3→7→14일로 벌리며 3번(어려운 관용구는 4번) 통과해야 졸업**시켜 까먹기 직전 복습으로 기억을 굳히고(8-1), 한 단어를 **뜻·철자·발음 세 점수로 분리 기록**해 맞춤 학습의 토대를 놓았으며(8-2), 신규:복습 비율을 **복귀자·초보·오답 많은 사람마다 자동 조정**하고(8-3), 레벨이 올라도 **이전 레벨 약점 단어를 계속 복습**하게 했습니다(8-4). 레벨 조정은 한 판의 운이 아닌 **최근 3~5세트 누적 성적표**로 판단해 흔들림을 없앴고(8-5), 졸업한 단어도 **한 달 뒤 슬쩍 재점검**해 장기 망각을 막습니다(8-6). 모든 두뇌 로직은 화면·데이터와 분리해 **자동 테스트 33개(Phase 8 25 + SRS 8)로 검증**(전부 통과)했고, 클라우드 저장은 **마이그레이션 전에도 멈추지 않는 무중단 장치**를 넣었습니다.
>
> **결과: MVP 로드맵(Phase 0~8) 전 구간이 완성되었습니다. 학습 루프 → 콘텐츠 품질 → 진단·채점 신뢰도 → 장기 기억·적응까지, 하나의 학습 시스템으로 이어집니다.**
